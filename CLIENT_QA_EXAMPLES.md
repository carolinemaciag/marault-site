## Key Response Patterns Demonstrated

### Pattern 1: Size Reassurance
**When:** User mentions "small", "startup", "enterprise"  
**Response:** Emphasize all sizes welcome, mindset matters more  
**Effect:** Removes barriers, invites conversation

### Pattern 2: Industry Validation
**When:** User mentions specific industry  
**Response:** Confirm experience in that industry, note common patterns  
**Effect:** Shows expertise, builds confidence

### Pattern 3: Role Recognition
**When:** User mentions role (CEO, CFO, etc.)  
**Response:** Name the role, acknowledge decision-making authority  
**Effect:** Makes user feel understood and relevant

### Pattern 4: Fit Uncertainty
**When:** User questions if they're a fit  
**Response:** Redirect to mindset/values rather than metrics  
**Effect:** Opens door wider, less exclusionary

### Pattern 5: Problem-Focused
**When:** User mentions specific challenge  
**Response:** Acknowledge problem, connect to service  
**Effect:** Moves conversation toward solutions

## Response Characteristics

### ✅ Always Present
- Welcoming tone (no gatekeeping)
- Specific examples (industries, roles, sizes)
- Values-focused (clarity, decisions, structure)
- Action-oriented (suggest next step or service)

### ❌ Never Present
- "We don't work with..."
- Size limitations
- Industry exclusions
- Requirement gatekeeping
- Negative qualifications

### 🎯 Effective Elements
- Specific industry/role naming (feels understood)
- Mindset-first messaging (inclusive)
- Common thread emphasis (clarity unites all)
- Service pathway (moves toward solution)

## Production Notes

All these Q&A examples are generated from the 4 handlers:
1. Main WHO DO YOU WORK WITH (general fit)
2. COMPANY SIZE & STAGE (size concerns)
3. SPECIFIC INDUSTRIES (industry fit)
4. SPECIFIC ROLES/LEADERSHIP (role fit)

Handlers route based on keywords in the question. The router (`contains()` function) selects the most specific handler match.
